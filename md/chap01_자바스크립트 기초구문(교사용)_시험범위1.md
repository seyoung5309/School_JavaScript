# 1. 자바스크립트란

- **동적인 웹 사이트 제작을 위해 사용**됨

1. 역사 : 1990년대 넷스케이프의 브랜던 아이크(Brenden Eich)가 Mocha라는 이름으로 출시하고, 라이브스크립트로 개발하였다가 최종적으로 자바스크립트가 됨. 그러나 브라우져(넷스케이프, 익스플로러....)에 따라 상이한 스크립트가 나오기 시작하면서 개발이 쉽지 않았으며(if 구문의 남발...).... 스크립트의 표준화에 대한 요구가 생기기 시작함.

2. ECMA(European Computer Manufactures Association) 스크립트 : 자바스크립트의 표준작업이 시작된 것으로 1997년 6월 정식으로 채택된 스크립트의 표준. 모던 스크립트는 ES(ECMA script의 약자)6 버전에서 시작한다고 본다.

3. 바닐라 자바스크립트(바닐라 JS) : 어떠한 라이브러리(JQeury..)나 프레임워크(React, Vue, Angular....)를 사용하지 않는 순수 자바스크립트, 가장 빠르고 가벼운 크로스 플랫폼 프레임워크 => 실력 향상을 위한 필수 단계 => 결코 쉽지 않다...=> 그러나 프레임워크나 라이브러리는 모두 바닐라 자바스크립트로 만들어져 있다. => 자신의 실력과 가치를 높이는 강력한 수단이 될 수 있다.

4. 바닐라 자바스크립트 시작하기

- 사이트 : http://vanilla-js.com/
- 시작 : <script src="path/to/vanilla.js"></script>를 html 문서안에 삽입
- 위트 있는 사이트 ^^ : 다운받아도 아무 것도 없다!!!!
- 순수 자바스크립트로 짜면 좋다!!!!

- 최근에는 자바스크립트를 이용하여 웹뿐만 아니라 서버(NodeJS), 데스크톱 프로그램(Electron)도 제작하고 있는 추세
- 최근 [가장 인기있는 언어](http://githut.info/)로 높은 점유율을 차지하고 있음

# 4. 변수 선언자 var, let, const

- 변수란? 프로그램 실행에 필요한 데이터를 담는 기억공간
  - 장점 : 데이터 타입에 따른 선언(int, float, double.... )이 필요하지 않다.
  - 단점 : 데이터 타입에 관계없는 변수이기 때문에 개발시 오류 발생이 가능

- 변수 선언자 종류 : : var, (ES6부터)let, const
- 선언 형식 : 선언자 변수명 = 값;
- 변수 작성 규칙(4가지) :

  ① 카멜 표기법(camelCase) : 두 가지 이상의 단어가 합쳐질 경우, 두 번째부터 단어의 첫 글자를 대문자로

      var userName = ‘John Doe’;

  ② 스네이크 표기법(snake_case) : 두 개 이상의 단어가 합쳐질 경우 단어 사이에 \_ 삽입

      var user_name = ‘John Doe’;

  ③ 파스칼 표기법(PascalCase): 모든 단어의 첫 글자를 대문자로

      var UserName = ‘John Doe’;

  ④ 헝가리안 표기법 : 변수명 맨 앞에 데이터의 타입에 해당하는 영어단어를 추가하고, 나머지는 카멜 표기법을 따른다.

      var strUserName = “John Doe”;

- 최근의 추세는 몇 번? 자바스크립트는 ① 카멜 표기법을 사용한다.

1. var 선언자의 특징
   - ES6 이전부터 사용한 선언자
   - 재선언(동일한 이름으로 재선언이 가능), 재할당 가능
   - 재선언이 가능한 변수 => 협업을 하면서 동일한 이름의 변수를 var로 선언할 경우 덮어쓰기가 가능해서 예상하지 않은 결과가 나올 수 있다.
   - 전역변수로 사용 가능

2. **let** 선언자의 특징
   - ES6 이후부터 사용한 선언자
   - 재선언이 불가능한 변수 선언 => 동일한 변수명을 다른 곳에서 사용 불가 => 권장
   - 재할당은 가능

3. **const** 선언자의 특징
   - ES6 이후부터 사용한 선언자
   - 변하지 않는 상수값 선언에 사용
   - 상수는 대문자로 표기한다.
   - 재선언, 재할당 모두 불가능하다. (재할당을 해도 에러가 발생하지 않고 예외가 발생한다. )
   - 협업하는 개발자들이 통일해서 사용하는 데이터 포맷 선언 등에 사용한다. 예를 들어 개발 중인 사이트에서 사용하는 날짜의 포맷을 const dateFormat=’YYYY.MM.DD’; 으로 통일하는 경우 등이 해당된다.

# 6. 데이터 타입 - Object

- 정의 : 관련 있는 데이터를 하나로 담을 수 있는 객체
  - 파이썬의 딕셔너리와 매우 유사하다.
  - 생성 후 데이터를 자유롭게 수정, 추가, 삭제가 가능하다.
  - 사용법 : { }를 사용
  - {key : value} 형태를 가진다.
- 특징 : 관리가 매우 쉽다.

1. 관련 있는 정보를 하나로 묶어서 관리하고 싶은 경우 사용 : 사람 – 이름, 전화번호, 이메일주소, 집 주소 등을 묶고 싶은 경우

2. 키(key : name, phone, email 등 ....) 와 값(value)를 쌍으로 갖는다.

let person = {

    name : "홍길동",

    phone : "010-0000-0000",

    email : "hong@gmail.com",

    address : "서울시 강서구 화곡동"

};

3. 객체의 값에 접근하는 방법(.연산자 사용) : 이름.키

   let name = person.name;

4. 객체의 값을 변경시 : 키 값을 이용

   person.address = "제주시"

5. 객체에 새로운 키와 값을 추가하는 방법

   person.bloodType = "B";

6. object 의 특정 키에 접근하는 방법 두 가지
   - 변수명.키
   - 변수명[\"키\"]
   - 둘의 차이점?? [ ]를 사용하는 경우 동적(실행시간, 런타임)으로 특정 키를 가져와야 할 경우 활용 가능하다.

   - 만약 존재하지 않는 키에 접근하고자 하는 경우 어떤 일이 발생하는가?

     당연히 undefined 값이 출력된다.

# 9. 연산자

1. 할당 연산자

- 특징 : 오른쪽 피연산자의 값을 왼쪽 변수에 할당
- 종류 : =, 복합 할당 연산자 (+=, -=, \*=, /=, %= )

2. 비교 연산자

- 특징 : 값(타입) 비교
  - 다음의 경우 데이터 타입이 달라도 값이 같게 나오므로 주의해야 한다. (언어의 유연성)

    x = 3;

    y = "3";

    console.log(x == y); //true

- 종류 : ( ==. ===, !=, !==, >, >=, <, <= )
- 값과 타입의 비교 : ===, !==

3. 산술 연산자

- 특징 : 증가, 감소, 단항 부정(-), 숫자화(+)
- 종류 : +, -, \*, /, %(나머지) , \*\*(제곱근), ++, --

  3-1. 단항 부정 연산자
  - 특징 : 부호가 바뀜

  3-2. 숫자화 연산자
  - 특징 : 숫자가 아닌 문자/boolean 값을 숫자로 변환하는 연산자.

4. 논리 연산자

- 특징 : 여러 가지 조건을 만족시키는지 알아보는 연산자
- 종류 : &&(둘 다 true 일 경우 true), ||(둘 중 하나만 true 이면 true)

5. 문자열 관련 연산자

- 특징 : 합치기
- 종류 : +

6. 조건(삼항) 연산자

- 특징
  - 조건 ? true 일 때 할당할 값 : false 일 때 할당할 값
  - 중첩해서 사용이 가능하다.

# 10. 조건문

- 특징
  - 가장 중요한 구문 중 하나이다.
  - 프로그램 = 조건문의 집합
  - if : 만일 ~ 라면...
  - 작성 시 순서가 효율성에 영향을 미친다.

1. 조건문 if

- if :

  if (조건식) {

  조건을 만족할 경우 실행할 코드 블록

  }

- if else :
  if(조건식) {

        조건을 만족할 경우 실행할 코드 블록

  }else {

        조건을 만족하지 않을 경우 실행할 코드 블록

  }
